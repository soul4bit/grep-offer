# Docker, CI/CD и момент, когда "локально работало" заканчивается

На этом этапе важно понять, что доставка кода это не ритуал, а цепочка из конкретных шагов.

## Что нужно собрать

- Dockerfile без слоя магии
- понятный pipeline с тестами и сборкой
- healthcheck и rollback, чтобы прод не жил на силе веры

## Минимум руками

1. Собрать контейнер с приложением.
2. Запушить образ в registry.
3. Поднять release на сервере.
4. Проверить `healthz` и откатить, если стало хуже.

```bash
docker build -t grep-offer:local .
docker run --rm -p 8080:8080 grep-offer:local
curl http://127.0.0.1:8080/healthz
```

## Сигнал, что все идет нормально

После деплоя ты умеешь ответить на три вопроса: что поехало, куда поехало и как это откатить без драматического поста в чат.
