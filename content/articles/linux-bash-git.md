# Linux, bash и git без шаманства

Сначала надо перестать бояться терминала. Не любить, не романтизировать, а просто перестать бояться.

## Что здесь должно появиться в голове

- как ходить по файловой системе и не теряться
- что делают `ps`, `top`, `systemctl`, `journalctl`
- почему `grep`, `sed`, `awk`, `find` и пайпы экономят часы
- чем `git fetch` отличается от реальности в голове после бессонной ночи

## Минимум руками

1. Поднять локальный nginx.
2. Посмотреть его логи.
3. Уронить конфиг.
4. Найти проблему через логи и вернуть все обратно.

```bash
sudo systemctl status nginx
sudo journalctl -u nginx -n 50 --no-pager
ss -tulpn | grep 80
```

## Как понять, что этап не мимо

Если приложение умерло, ты сначала смотришь в процесс, порт и лог, а не в луну и ретроградный YAML.
